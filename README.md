# 智慧职教脚本使用说明

> 仅用于浏览器脚本学习、个人调试和代码维护。使用前请确认自己的使用场景符合学校、课程和平台规则。

## 支持平台

当前入口脚本会根据浏览器地址自动识别平台：

| 平台 | 地址 | 加载的核心脚本 |
| --- | --- | --- |
| 职教云 | `https://zjy2.icve.com.cn` | `main/cont.js` |
| 智慧职教 | `https://mooc.icve.com.cn` | `main/cont.js` |
| 职教云资源库 | `https://www.icve.com.cn` | `main/special_cont.js` |

如果在其他域名执行脚本，会弹出提示，可输入：

- `1`：跳转到职教云
- `2`：跳转到智慧职教
- `3`：跳转到职教云资源库
- 其他内容：取消跳转

## 功能总览

脚本加载成功后，会在页面左侧、右侧和中间区域插入一个控制面板，用来完成平台切换、课程选择、运行控制、速度设置、课程视图查看和缓存管理。

主要功能：

| 功能 | 说明 |
| --- | --- |
| 平台识别 | 根据当前域名自动判断职教云、智慧职教或职教云资源库。 |
| 平台跳转 | 在非支持域名执行时，可按提示跳转到支持平台。 |
| 自动加载依赖 | 入口脚本会检测页面是否存在 jQuery；没有时会自动加载 jQuery 后再加载核心脚本。 |
| 用户信息读取 | 登录后读取用户名、学号、头像等信息，并显示在脚本面板左侧。 |
| 课程列表读取 | 读取当前账号的课程列表，并显示可选择课程。 |
| 课程运行控制 | 通过“运行/暂停”按钮启动或暂停脚本流程。 |
| 课程节点处理 | 按课程、模块、章节、子节点顺序处理文档、视频、音频等内容。 |
| 课程视图 | 生成课程结构视图，可查看课程层级并快速定位未完成节点。 |
| 跳过控制 | 支持跳过文档、跳过视频/音频、跳过当前子节点。 |
| 速度设置 | 可设置请求发送速度、文档修改速度、视频修改速度、视频进度增加值。 |
| 进度缓存 | 会把部分课程列表和运行状态保存到浏览器 `localStorage`，下次运行时可继续使用。 |
| 已完成课程切换 | 可在未完成课程和已完成课程列表之间切换查看。 |
| 背景设置 | 支持在线图片链接或本地图片作为面板背景。 |
| 日志输出 | 脚本运行过程会在面板中输出状态、错误和当前处理节点。 |
| 清除缓存 | 运行异常时可清除脚本缓存并刷新页面重新执行。 |

平台差异：

- 职教云、智慧职教使用 `main/cont.js`。
- 职教云资源库使用 `main/special_cont.js`。
- 作业、习题、测验、考试类内容不会自动提交；脚本会提示人工处理，并从本次自动处理队列中移除。

## 脚本运行原理

整体流程分为入口加载、平台识别、用户初始化、课程解析、节点处理、状态缓存和界面反馈。

### 1. 入口加载

`main/app.js` 是唯一推荐入口。

执行后会做三件事：

1. 根据 `location.origin` 判断当前平台。
2. 检查页面是否已有 `window.jQuery`。
3. 如果没有 jQuery，先加载 `jquery@3.6.0`，再加载对应核心脚本。

加载规则：

```text
职教云、智慧职教    -> main/cont.js
职教云资源库        -> main/special_cont.js
```

### 2. 用户初始化

核心脚本加载后，会向平台接口请求当前用户信息。

成功后：

- 在左侧显示头像、用户名、学号。
- 在中间日志区域输出欢迎信息。
- 解锁“运行”按钮。

失败时：

- 提示先登录。
- 延迟刷新页面。

### 3. 课程列表解析

点击“运行”后，脚本会请求课程列表接口。

处理步骤：

1. 获取课程列表。
2. 根据进度筛选未完成课程。
3. 已完成课程放入 `finishList`。
4. 未完成课程放入 `CourseList`。
5. 把课程卡片渲染到课程列表区域。
6. 开启课程选择倒计时。

### 4. 模块、章节、子节点解析

选中课程后，脚本按层级继续请求：

```text
课程 -> 模块 -> 章节 -> 子节点
```

解析后的未完成子节点会放入 `unNodeList`，每个节点保存一个索引，例如：

```text
模块索引-章节索引-子节点索引
```

脚本运行时就是按 `unNodeList` 顺序逐个处理。

### 5. 文档、视频、音频处理

普通学习内容会进入进度处理流程。

文档类：

- 按配置的“文档修改速度”发送进度请求。
- 完成后标记为已完成。

视频/音频类：

- 按配置的“视频修改速度”循环发送进度请求。
- 每次把当前进度增加“视频修改时间”设置的秒数。
- 如果平台返回异常，会降速重试。
- 连续异常过多会暂停或跳过当前节点。

### 6. 作业、习题、测验、考试处理

当前版本不会自动提交作业、习题、测验或考试。

处理原则：

1. 普通平台中，测验类节点会被过滤，不进入自动处理队列。
2. 资源库中，如果识别到 `work` 或 `question` 节点，会输出人工处理提示。
3. 该节点会从本次自动处理队列中移除。
4. 脚本不会提交答案，也不会修改平台真实完成状态。
5. 课程视图中这类节点会显示为橙色，表示需要人工处理。

### 7. 状态缓存

脚本使用 `localStorage` 保存部分状态。

主要键名：

| 键名 | 用途 |
| --- | --- |
| `s_bg` | 面板背景图 |
| `s_courseList` | 课程列表和节点缓存 |
| `scriptID` | 脚本版本和缓存标记 |

缓存的作用是减少重复请求，让刷新后可以较快恢复课程结构。但如果课程列表错乱、换账号或平台数据变化，应该点击“清除缓存”。

### 8. 图形化界面原理

核心脚本会动态插入一整套 HTML 和 CSS：

- 左侧：账号信息、平台切换、运行/暂停。
- 中间：日志、课程列表、课程视图、背景设置。
- 右侧：速度配置、功能菜单、缓存操作。

课程视图颜色：

| 颜色 | 含义 |
| --- | --- |
| 绿色 | 脚本已处理或平台已完成 |
| 橙色 | 作业、习题、测验、考试，需要人工处理 |
| 灰色 | 未处理或普通待处理节点 |

## 方式一：使用线上入口

适合已经把本仓库代码发布到 GitHub，并希望直接从 CDN 加载的情况。注意：jsDelivr 只能直接访问公开仓库，私有仓库不能使用下面这种 CDN 地址。

1. 使用 Chrome 或 Edge 打开目标平台。
2. 登录账号，确认页面已经进入平台内部，不要停留在未登录页面。
3. 按 `F12` 打开开发者工具。
4. 切换到 `Console`（控制台）。
5. 如果控制台提示禁止粘贴，按提示输入 `allow pasting` 后回车。
6. 粘贴下面代码并回车执行：

```js
var script = document.createElement("script");
script.src = "https://fastly.jsdelivr.net/gh/liuhuihou/auto-play-course-main@main/main/app.js";
document.body.appendChild(script);
```

7. 等待页面出现脚本面板。
8. 点击面板里的“运行”按钮开始执行。
9. 需要暂停时，再次点击按钮，按钮会切换为“暂停/运行”状态。

如果你刚修改了 `main/app.js`，但线上执行仍然是旧逻辑，通常是 CDN 缓存未刷新。可以临时给地址加版本号：

```js
var script = document.createElement("script");
script.src = "https://fastly.jsdelivr.net/gh/liuhuihou/auto-play-course-main@main/main/app.js?v=20260708";
document.body.appendChild(script);
```

更稳定的做法是把 `@main` 改成具体 commit hash，避免 CDN 分支缓存导致版本不确定。

## 方式二：本地调试入口

适合你刚修改了本地代码，还没有发布到 GitHub/CDN 的情况。

### 方法 A：直接粘贴本地入口代码

1. 打开 `main/app.js`。
2. 复制文件里的全部代码。
3. 登录目标平台。
4. 按 `F12` 打开开发者工具。
5. 进入 `Console`。
6. 粘贴 `main/app.js` 的完整代码并回车。

这种方式可以立刻验证本地入口脚本的改动。注意：当前 `main/app.js` 会从 CDN 加载 `cont.js` 或 `special_cont.js`。

### 方法 B：通过本地静态服务加载

1. 在项目根目录启动静态服务：

```bash
python -m http.server 8000
```

2. 登录目标平台。
3. 打开开发者工具控制台。
4. 粘贴下面代码：

```js
var script = document.createElement("script");
script.src = "http://127.0.0.1:8000/main/app.js";
document.body.appendChild(script);
```

5. 如果浏览器拦截本地 HTTP 脚本加载，改用“方法 A：直接粘贴本地入口代码”。

## 标准使用流程

推荐按下面顺序使用：

1. 打开支持的平台地址。
2. 登录账号，确认页面已经进入平台内部。
3. 打开浏览器控制台。
4. 执行 README 中的线上入口代码，或直接粘贴本地 `main/app.js` 代码。
5. 等待脚本面板出现。
6. 查看左侧是否显示用户名、学号和头像。
7. 查看中间日志区域，确认没有红色报错。
8. 点击“运行”。
9. 首次运行会加载课程列表，页面会提示在限定时间内选择课程。
10. 在课程列表中点击要处理的课程；如果不选，脚本会按默认逻辑继续。
11. 运行过程中通过中间日志观察当前课程、章节、节点和异常提示。
12. 需要暂停时点击“暂停”。
13. 需要继续时再次点击“运行”。
14. 需要换课时点击“更换课程”，重新选择课程。
15. 运行异常或缓存混乱时点击“清除缓存”，确认后页面会刷新，刷新后重新执行脚本。

推荐首次使用时不要修改右侧速度配置，先使用默认值观察是否能稳定运行。确认稳定后再按需要调整速度。

## 面板功能详解

脚本面板主要分为左侧信息栏、中间内容区和右侧设置栏。

### 左侧信息栏

| 区域 | 功能 | 使用方法 |
| --- | --- | --- |
| 用户头像 | 显示当前登录账号头像。 | 登录后由平台接口读取。 |
| 用户名 | 显示当前账号显示名。 | 如果为空，先确认是否已登录。 |
| 学号 | 显示当前账号学号或用户名。 | 如果读取失败，可刷新页面后重新执行脚本。 |
| 切换平台 | 打开平台切换列表。 | 点击后选择职教云、智慧职教或职教云资源库。 |
| 运行/暂停 | 启动或暂停脚本。 | 未运行时按钮显示“运行”，运行中显示“暂停”。 |

左侧边缘有展开/收起按钮，可折叠左侧信息栏。

### 中间内容区

| 区域 | 功能 | 使用方法 |
| --- | --- | --- |
| 日志区域 | 显示脚本运行状态、当前节点、异常信息。 | 运行时优先看这里判断脚本是否正常。 |
| 课程列表 | 显示可选择的课程。 | 点击课程卡片后脚本会切换到该课程并开始处理。 |
| 已完成课程切换 | 在未完成课程和已完成课程之间切换。 | 点击“切换为已完成课程”或“切换为未完成课程”。 |
| 课程视图 | 显示课程层级结构。 | 点击“打开视图”后查看章节和子节点。 |
| 背景设置 | 设置面板背景图。 | 点击“更换背景”，输入图片 URL 或选择本地图片。 |
| 支持作者 | 展示支持信息。 | 点击“支持作者”打开。 |

课程列表第一次出现时，顶部会显示倒计时提示。倒计时期间可以手动点选课程；未选择时脚本会按默认流程继续。

### 右侧设置栏

右侧设置栏用于调整速度和打开功能菜单。右侧边缘也有展开/收起按钮。

速度设置：

| 设置项 | 默认值 | 可填范围 | 说明 |
| --- | --- | --- | --- |
| 请求发送速度 | `3` 秒 | `1-4` | 控制接口请求和流程推进间隔。 |
| 文档修改速度 | `2` 秒 | `1-4` | 控制文档类节点的处理间隔。 |
| 视频修改速度 | `10` 秒 | `8-15` | 控制视频/音频类节点的请求间隔，下一个视频后生效。 |
| 视频修改时间 | `15` 秒 | `12-22` | 每次把视频当前进度向前增加的秒数，下一个视频后生效。 |

修改方式：

1. 点击输入框。
2. 输入范围内的数字。
3. 鼠标移出输入框或让输入框失焦。
4. 日志区域出现“修改成功”提示后生效。

速度设置过快可能导致平台接口异常或检测异常，建议优先使用默认值。

### 功能菜单

| 按钮 | 作用 | 使用时机 |
| --- | --- | --- |
| 更换课程 | 打开课程列表，重新选择课程。 | 需要切换课程或重新加载课程时使用。 |
| 跳过文档 | 开启或关闭文档跳过模式。 | 当前不想处理文档类节点时使用。 |
| 跳过视频 | 开启或关闭视频/音频跳过模式。 | 当前不想处理视频或音频节点时使用。 |
| 跳过本节 | 只跳过当前正在处理的子节点。 | 当前节点异常、卡住或不想处理时使用。 |
| 打开视图 | 打开课程结构视图。 | 需要查看课程层级或跳转到指定未完成节点时使用。 |
| 反馈意见 | 打开反馈链接。 | 需要反馈问题时使用。 |
| 支持作者 | 打开支持作者信息。 | 查看支持信息时使用。 |
| 更换背景 | 打开背景设置面板。 | 需要自定义面板背景时使用。 |
| 清除缓存 | 清除脚本本地缓存并刷新页面。 | 脚本异常、课程列表错乱、进度缓存异常时使用。 |
| 点我清除页面信息 | 清空中间日志区域。 | 日志过多时使用，不会清除运行缓存。 |

注意：

- “跳过文档”“跳过视频”“跳过本节”需要脚本已经开始运行后再使用。
- “跳过文档”和“跳过视频”是互斥模式，开启一个会关闭另一个。
- “跳过本节”是一次性操作，只跳过当前子节点，不会持续跳过后续节点。

### 课程视图用法

1. 点击“打开视图”。
2. 等待脚本构建课程结构。
3. 点击上级章节可展开或收起。
4. 点击未完成子节点，可让脚本切换到该节点继续处理。
5. 已完成节点会提示无需执行。

如果点击未完成节点时提示“请先运行脚本”，先点击左侧“运行”，再回到课程视图选择节点。

### 背景设置用法

支持两种方式：

1. 在线图片：点击“更换背景”，在 `URL` 输入框粘贴图片地址，点击“确定”。
2. 本地图片：点击“选择文件”，选择本地图片。

本地图片会以 Base64 形式保存到浏览器 `localStorage`，文件大小不能超过 2MB。

### 缓存说明

脚本会使用浏览器 `localStorage` 保存部分数据，包括：

- 背景图：`s_bg`
- 课程列表缓存：`s_courseList`
- 脚本版本/缓存标记：`scriptID`

什么时候需要清除缓存：

- 课程列表显示不对。
- 已完成/未完成课程切换异常。
- 脚本连续异常暂停。
- 修改了核心逻辑后仍表现像旧版本。
- 换账号后仍显示旧账号相关数据。

清除方式：

1. 点击“清除缓存”。
2. 在确认框中确认。
3. 页面刷新后重新执行入口脚本。

## 注意事项

1. 请先登录平台，再执行脚本。
2. 建议进入课程页面后再执行，避免平台接口还没有初始化完成。
3. 作业、习题、测验、考试类内容不会自动提交，需要在平台页面手动完成。
4. 脚本会在 `localStorage` 保存部分配置和进度。换浏览器、清缓存或使用无痕窗口会导致这些数据丢失。
5. 本地背景图会保存到浏览器本地存储中，图片过大可能导致保存失败，建议控制在 2MB 以内。
6. 如果连续异常，脚本可能自动暂停。刷新页面后重新执行即可。

## 常见问题

### 1. 控制台提示 `$ is not defined`

新版 `main/app.js` 已经增加 jQuery 兜底加载。出现这个错误通常说明你加载的还是旧版入口。

处理方式：

1. 确认线上 CDN 已经更新。
2. 给 CDN 地址加版本号。
3. 或直接把本地 `main/app.js` 内容粘贴到控制台执行。

### 2. 执行后没有出现面板

按下面顺序检查：

1. 当前域名是否为支持平台。
2. 是否已经登录。
3. 控制台是否有红色报错。
4. CDN 地址是否能访问。
5. 是否被浏览器插件、广告拦截器或网络策略拦截。

### 3. 修改了代码但浏览器仍然运行旧版本

可能是浏览器缓存或 CDN 缓存。

处理方式：

- 地址后追加 `?v=日期或版本号`。
- 强制刷新页面：`Ctrl + F5`。
- 清空浏览器缓存。
- 使用具体 commit hash 替代 `@main`。

### 4. 提示脚本加载失败

新版入口在脚本加载失败时会弹出失败地址。复制该地址到浏览器新标签页打开，确认是否能访问。

## 维护和验证

修改脚本后，至少执行下面检查：

```bash
node --check main/app.js
node --check main/cont.js
node --check main/special_cont.js
```

当前入口默认加载源码版 `main/cont.js` 和 `main/special_cont.js`，因此修改源码后推送即可生效。`.min.js` 文件保留为旧版备用，不建议继续作为默认入口。
