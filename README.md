# 智慧职教脚本使用说明

> 仅用于浏览器脚本学习、个人调试和代码维护。使用前请确认自己的使用场景符合学校、课程和平台规则。

## 支持平台

当前入口脚本会根据浏览器地址自动识别平台：

| 平台 | 地址 | 加载的核心脚本 |
| --- | --- | --- |
| 职教云 | `https://zjy2.icve.com.cn` | `main/cont.min.js` |
| 智慧职教 | `https://mooc.icve.com.cn` | `main/cont.min.js` |
| 职教云资源库 | `https://www.icve.com.cn` | `main/special_cont.min.js` |

如果在其他域名执行脚本，会弹出提示，可输入：

- `1`：跳转到职教云
- `2`：跳转到智慧职教
- `3`：跳转到职教云资源库
- 其他内容：取消跳转

## 文件说明

```text
main/app.js              入口脚本，负责识别平台、加载 jQuery、加载核心脚本
main/cont.js             职教云/智慧职教核心脚本源码
main/cont.min.js         职教云/智慧职教核心脚本压缩版
main/special_cont.js     职教云资源库核心脚本源码
main/special_cont.min.js 职教云资源库核心脚本压缩版
智慧职教刷课脚本.js       旧版 1.0 脚本，仅支持早期职教云
img/                     README 和脚本界面用到的图片资源
```

## 方式一：使用线上入口

适合已经把本仓库代码发布到 GitHub，并希望直接从 CDN 加载的情况。

1. 使用 Chrome 或 Edge 打开目标平台。
2. 登录账号，确认页面已经进入平台内部，不要停留在未登录页面。
3. 按 `F12` 打开开发者工具。
4. 切换到 `Console`（控制台）。
5. 如果控制台提示禁止粘贴，按提示输入 `allow pasting` 后回车。
6. 粘贴下面代码并回车执行：

```js
var script = document.createElement("script");
script.src = "https://fastly.jsdelivr.net/gh/hsjzhcq/hcqHome@main/main/app.js";
document.body.appendChild(script);
```

7. 等待页面出现脚本面板。
8. 点击面板里的“运行”按钮开始执行。
9. 需要暂停时，再次点击按钮，按钮会切换为“暂停/运行”状态。

如果你刚修改了 `main/app.js`，但线上执行仍然是旧逻辑，通常是 CDN 缓存未刷新。可以临时给地址加版本号：

```js
var script = document.createElement("script");
script.src = "https://fastly.jsdelivr.net/gh/hsjzhcq/hcqHome@main/main/app.js?v=20260708";
document.body.appendChild(script);
```

更稳定的做法是把 `@main` 改成具体 commit hash，避免 CDN 分支缓存导致版本不确定。

## 方式二：本地调试入口

适合你刚修改了本地代码，还没有发布到 GitHub/CDN 的情况。

### 方法 A：直接粘贴本地入口代码

1. 打开 `main/app.js`。
2. 复制文件里的全部代码。
3. 登录目标平台。
4. 按 `F12` 打开开发者工具。
5. 进入 `Console`。
6. 粘贴 `main/app.js` 的完整代码并回车。

这种方式可以立刻验证本地入口脚本的改动。注意：当前 `main/app.js` 仍会从 CDN 加载 `cont.min.js` 或 `special_cont.min.js`。

### 方法 B：通过本地静态服务加载

1. 在项目根目录启动静态服务：

```bash
python -m http.server 8000
```

2. 登录目标平台。
3. 打开开发者工具控制台。
4. 粘贴下面代码：

```js
var script = document.createElement("script");
script.src = "http://127.0.0.1:8000/main/app.js";
document.body.appendChild(script);
```

5. 如果浏览器拦截本地 HTTP 脚本加载，改用“方法 A：直接粘贴本地入口代码”。

## 面板操作说明

脚本加载成功后，会在页面中插入控制面板。

常用按钮：

- “运行”：开始执行脚本。
- “暂停”：暂停当前执行流程。
- “切换平台”：切换职教云、智慧职教、职教云资源库。
- “更换课程”：重新选择或刷新课程列表。
- “跳过文档”：跳过文档类节点。
- “跳过视频”：跳过视频/音频类节点。
- “跳过本节”：跳过当前子节点。
- “打开视图”：打开课程结构视图，可快速定位章节。
- “更换背景”：设置面板背景图。
- “清除缓存”：清除脚本保存在浏览器本地的进度缓存。

课程列表底部有“切换为已完成课程”按钮，可在未完成课程和已完成课程之间切换。

## 注意事项

1. 请先登录平台，再执行脚本。
2. 建议进入课程页面后再执行，避免平台接口还没有初始化完成。
3. 智慧职教部分测验题不支持自动处理，遇到测验题时可能需要手动完成。
4. 脚本会在 `localStorage` 保存部分配置和进度。换浏览器、清缓存或使用无痕窗口会导致这些数据丢失。
5. 本地背景图会保存到浏览器本地存储中，图片过大可能导致保存失败，建议控制在 2MB 以内。
6. 如果连续异常，脚本可能自动暂停。刷新页面后重新执行即可。

## 常见问题

### 1. 控制台提示 `$ is not defined`

新版 `main/app.js` 已经增加 jQuery 兜底加载。出现这个错误通常说明你加载的还是旧版入口。

处理方式：

1. 确认线上 CDN 已经更新。
2. 给 CDN 地址加版本号。
3. 或直接把本地 `main/app.js` 内容粘贴到控制台执行。

### 2. 执行后没有出现面板

按下面顺序检查：

1. 当前域名是否为支持平台。
2. 是否已经登录。
3. 控制台是否有红色报错。
4. CDN 地址是否能访问。
5. 是否被浏览器插件、广告拦截器或网络策略拦截。

### 3. 修改了代码但浏览器仍然运行旧版本

可能是浏览器缓存或 CDN 缓存。

处理方式：

- 地址后追加 `?v=日期或版本号`。
- 强制刷新页面：`Ctrl + F5`。
- 清空浏览器缓存。
- 使用具体 commit hash 替代 `@main`。

### 4. 提示脚本加载失败

新版入口在脚本加载失败时会弹出失败地址。复制该地址到浏览器新标签页打开，确认是否能访问。

## 维护和验证

修改脚本后，至少执行下面检查：

```bash
node --check main/app.js
node --check main/cont.min.js
node --check main/special_cont.min.js
```

如果修改的是源码文件 `main/cont.js` 或 `main/special_cont.js`，需要同步更新对应的 `.min.js` 文件，否则线上入口仍然会加载旧的压缩版逻辑。
